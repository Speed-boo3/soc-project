# Incident Response Playbook

Based on the NIST SP 800-61 incident response lifecycle. This playbook covers the most common alerts generated by this project.

---

## The four phases

```
Preparation → Detection & Analysis → Containment & Recovery → Post-Incident
```

**Preparation** — monitoring is running, team knows their roles, playbook is up to date.

**Detection and Analysis** — an alert fires. First job is to confirm if it is real or a false positive.

**Containment, Eradication, Recovery** — stop the spread, remove the threat, restore normal operations.

**Post-Incident** — write a report, update rules if needed.

---

## Playbook: SSH Brute Force (RULE-001 / T1110)

1. Identify the source IP from the alert
2. Count total failed attempts and the time window
3. If more than 10 attempts in 5 minutes → block the IP at the firewall
4. Check if any attempt succeeded after the failures
5. If a login succeeded → escalate, treat as full compromise
6. If no login succeeded → document the IP, close the incident
7. Add IP to blocklist and update threat intel

---

## Playbook: Invalid User Login (RULE-002 / T1078)

1. Check if the username targeted is a real account
2. Look for other attempts from the same IP
3. If the IP has multiple attempts → possible username enumeration, block it
4. If isolated → log and monitor

---

## Playbook: HTTP Credential Stuffing (RULE-004 / T1110.004)

1. Identify the source IP and number of 401 responses
2. Check if any attempt returned 200 (successful login)
3. If successful login found → notify the account owner, reset credentials
4. If no success → rate-limit or block the IP at the WAF

---

## Playbook: Segfault (RULE-006 / T1203)

1. Identify which process crashed
2. Check if the crash followed a network request
3. Pull core dump if available
4. Check if the crash is repeatable — could be an exploitation attempt
5. Notify system owner and patch or isolate the affected service

## Severity levels

| Level    | Response time | Who to notify     |
|----------|--------------|-------------------|
| Low      | 24 hours     | Log only          |
| Medium   | 4 hours      | SOC team          |
| High     | 30 minutes   | SOC + management  |
| Critical | Immediate    | All stakeholders  |
